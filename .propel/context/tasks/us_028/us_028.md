# User Story - US_028

## Story ID
* ID: US_028

## Story Title
* Session Resume Capability

## Description
* As a nurse, I want to resume incomplete sessions from the last unanswered question, so that I don't have to restart the entire conversation when I'm interrupted.

## Acceptance Criteria
1. **Given** I have an incomplete session, **When** I open the treatment page, **Then** the system detects incomplete session state and displays confirmation dialog "You have an incomplete session. Would you like to resume?" with Resume and Start Over buttons.

2. **Given** I tap Resume, **When** the session resumes, **Then** the system loads partial form data from MMKV, prefills all previously answered fields, and starts AI conversation from the first unanswered question (lastFieldIndex + 1).

3. **Given** I tap Start Over, **When** the confirmation is accepted, **Then** the system clears partial session data from MMKV, resets lastFieldIndex to 0, and starts fresh conversation from first question.

4. **Given** session resumes, **When** AI asks the next question, **Then** the system preserves all previously answered field values and does NOT re-ask completed questions.

5. **Given** session is older than 24 hours, **When** I attempt to resume, **Then** the system displays warning "This session is expired. Starting new session." and automatically clears stale session data.

## Edge Cases
* What happens when lastFieldIndex is corrupted? (Default to index 0; start from beginning with warning "Unable to determine last position. Starting from beginning.")
* How does the system handle resume during network interruption? (Allow resume for local data entry; defer Azure OpenAI calls until connectivity restored)
