# User Story - US_027

## Story ID
* ID: US_027

## Story Title
* Session State Persistence

## Description
* As a developer, I want session state persistence with last field index tracking in MMKV, so that partial form data is saved automatically and can be resumed after app interruptions.

## Acceptance Criteria
1. **Given** form data is entered, **When** each field is completed, **Then** the system automatically saves the field value and last field index to MMKV session state without user action.

2. **Given** AI session is active, **When** the app is backgrounded, **Then** the system saves current session state including formData object, lastFieldIndex, and timestamp to MMKV before suspension.

3. **Given** app crashes or closes, **When** the crash occurs, **Then** the system has already persisted the last completed field data to MMKV with session recovery metadata.

4. **Given** session state is saved, **When** I inspect MMKV storage, **Then** the system stores TreatmentSession object with sessionId, patientMrn, formData (all completed fields), lastFieldIndex (0-14), completionStatus (in_progress), and timestamps.

5. **Given** multiple sessions exist, **When** I query session state, **Then** the system retrieves the most recent incomplete session for the current patient using sessionId and timestamp sorting.

## Edge Cases
* What happens when MMKV write fails during save? (Implement retry logic with 3 attempts; log error; display warning "Unable to save progress" if all retries fail)
* How does the system handle corrupted session state? (Validate JSON structure on read; discard corrupted state; start new session with warning message)
