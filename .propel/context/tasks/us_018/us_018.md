# User Story - US_018

## Story ID
* ID: US_018

## Story Title
* AI Voice Icon Component

## Description
* As a nurse, I want a visible AI voice icon with distinct visual states (active/inactive/listening), so that I can identify the current voice mode status at a glance.

## Acceptance Criteria
1. **Given** I view the treatment form, **When** the screen loads, **Then** the system displays the AI voice icon in a prominent location with clear visibility against the background.

2. **Given** voice mode is inactive, **When** I view the icon, **Then** the system shows the icon in inactive state (gray color, static) indicating voice is not active.

3. **Given** I activate voice mode, **When** I tap the icon, **Then** the system transitions to active state (blue color #1566A7, pulsing animation) within 200ms and starts listening.

4. **Given** voice is listening, **When** STT captures audio, **Then** the system displays listening state (animated waveform or ripple effect) with visual feedback synchronized to audio input.

5. **Given** I deactivate voice mode, **When** I tap the icon again, **Then** the system transitions to inactive state, stops listening, and removes all animations.

## Edge Cases
* What happens when voice icon is tapped during TTS playback? (Ignore tap during TTS; display tooltip "Please wait for question to complete")
* How does the system handle rapid icon taps? (Debounce taps with 500ms delay; prevent state flickering)
