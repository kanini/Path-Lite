# User Story - US_012

## Story ID
* ID: US_012

## Story Title
* Session Timeout Management

## Description
* As a nurse, I want automatic session timeout after 30 minutes of inactivity, so that my session is secured when I step away from the device.

## Acceptance Criteria
1. **Given** I am authenticated, **When** I interact with the app, **Then** the system resets the inactivity timer to 30 minutes and tracks the last activity timestamp.

2. **Given** 30 minutes pass without activity, **When** the inactivity timer expires, **Then** the system automatically logs out the user, clears the JWT token from storage, and redirects to the login screen.

3. **Given** I am near timeout, **When** 28 minutes of inactivity pass, **Then** the system displays a warning dialog "Your session will expire in 2 minutes. Continue working?" with Logout and Continue buttons.

4. **Given** I tap Continue, **When** the warning dialog appears, **Then** the system resets the inactivity timer to 30 minutes and dismisses the dialog.

5. **Given** I tap Logout, **When** the warning dialog appears, **Then** the system immediately logs out the user, clears session data, and redirects to login screen.

## Edge Cases
* What happens when the app is backgrounded during active session? (Pause inactivity timer when app backgrounds; resume timer when app returns to foreground)
* How does the system handle session timeout during form submission? (Save partial data before logout; display message "Session expired. Please log in to continue.")
