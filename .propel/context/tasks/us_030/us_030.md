# User Story - US_030

## Story ID
* ID: US_030

## Story Title
* Error Correction Navigation

## Description
* As a nurse, I want automatic navigation to erroneous fields during error correction, so that I can fix validation errors without manually searching for them.

## Acceptance Criteria
1. **Given** validation errors exist, **When** I respond "Yes, edit" to the edit prompt, **Then** the system detects edit intent using LLM classification and navigates to the first erroneous field automatically.

2. **Given** navigation occurs, **When** the system moves to the erroneous field, **Then** the system auto-focuses the field, scrolls it into view, and highlights it with error indicator (warning triangle icon in amber/orange).

3. **Given** field is focused, **When** navigation completes, **Then** the system re-asks the specific question for that field via TTS (e.g., "Let's correct the Date of Birth. What is the patient's date of birth?").

4. **Given** I provide corrected input, **When** I speak the correction, **Then** the system processes the response through STT → LLM cleaning → validation pipeline and updates the field if validation passes.

5. **Given** all errors in a section are corrected, **When** the last error is fixed, **Then** the system automatically returns to Review screen and triggers validation again.

## Edge Cases
* What happens when nurse declines to edit? (Return to Review screen; keep Continue button disabled; allow manual navigation to fields)
* How does the system handle new errors introduced during correction? (Add new errors to correction queue; navigate to newly erroneous fields after current field is fixed)
