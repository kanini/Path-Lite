# User Story - US_032

## Story ID
* ID: US_032

## Story Title
* Confirmation Screen Display

## Description
* As a nurse, I want a structured confirmation screen displaying all field values in organized sections, so that I can review all entered data before final submission.

## Acceptance Criteria
1. **Given** all mandatory fields are filled, **When** the form is complete, **Then** the system automatically redirects to Review accordion/tab and displays structured summary page with all field values.

2. **Given** I view the summary, **When** the screen loads, **Then** the system displays fields grouped by sections (Demographics & Admission, Clinical Intake) with section headers and clear visual separation.

3. **Given** I review field values, **When** I scan the summary, **Then** the system displays field labels and values in organized layout with mandatory fields marked with asterisk (*) and optional fields clearly differentiated.

4. **Given** navigation state is needed, **When** I'm on the Review screen, **Then** the system updates left panel navigation to highlight Review tab as active and shows patient summary (name, MRN, age, gender) in left panel.

5. **Given** I need to verify data, **When** I review the summary, **Then** the system provides clear visual indication of view mode with all fields displayed in read-only format initially.

## Edge Cases
* What happens when optional fields are empty? (Display "Not provided" or "-" for empty optional fields; maintain consistent layout)
* How does the system handle long field values? (Truncate with ellipsis; provide expand option for full value display)
