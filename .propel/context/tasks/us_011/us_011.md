# User Story - US_011

## Story ID
* ID: US_011

## Story Title
* JWT Authentication Service

## Description
* As a developer, I want a JWT-based authentication service that validates hardcoded mock credentials and generates session tokens, so that nurses can securely access the application with 30-minute session expiry.

## Acceptance Criteria
1. **Given** I need to validate credentials, **When** I implement the authentication service, **Then** the system validates username and password against hardcoded mock credentials (no database) and returns success/failure response.

2. **Given** authentication succeeds, **When** I generate a JWT token, **Then** the system creates a JWT with payload containing userId, username, role, and hospitalId, signed with JWT_SECRET, and sets expiry to 30 minutes.

3. **Given** a token is generated, **When** I store the token, **Then** the system saves the JWT token in MMKV secure storage and includes it in Authorization header for all API requests.

4. **Given** a token needs validation, **When** I verify the token, **Then** the system checks token signature, validates expiry timestamp, and rejects expired or invalid tokens with 401 Unauthorized response.

5. **Given** authentication fails, **When** invalid credentials are provided, **Then** the system returns error response "Invalid username or password" without revealing which field is incorrect for security.

## Edge Cases
* What happens when JWT_SECRET is not configured? (Throw configuration error on app startup; prevent authentication service initialization)
* How does the system handle token expiry during active session? (Implement token refresh mechanism; prompt user to re-authenticate when token expires)
