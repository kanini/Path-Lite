# User Story - US_016

## Story ID
* ID: US_016

## Story Title
* Speech-to-Text Integration

## Description
* As a nurse, I want voice capture with Speech-to-Text conversion, so that I can provide voice responses to form questions without manual typing.

## Acceptance Criteria
1. **Given** I need voice input, **When** I integrate @react-native-voice/voice library, **Then** the system has version 3.2+ installed with iOS and Android native speech recognition configured.

2. **Given** microphone access is needed, **When** the app requests permissions, **Then** the system prompts for microphone permission on first use and handles granted/denied states appropriately.

3. **Given** I activate voice input, **When** I tap the AI voice icon, **Then** the system starts listening, displays listening indicator, and captures audio with noise suppression enabled.

4. **Given** I speak a response, **When** STT processes my voice, **Then** the system converts speech to text in under 1 second and displays the captured text for verification.

5. **Given** STT completes, **When** transcription finishes, **Then** the system stops listening, removes listening indicator, and passes the transcribed text to LLM cleaning pipeline.

## Edge Cases
* What happens when microphone permission is denied? (Display error "Microphone access required for voice input. Enable in Settings."; provide fallback to manual text entry)
* How does the system handle background noise interference? (Configure noise suppression; display warning "Background noise detected. Please speak clearly." if confidence is low)
