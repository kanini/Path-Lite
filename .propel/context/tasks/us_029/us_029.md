# User Story - US_029

## Story ID
* ID: US_029

## Story Title
* Review Validation Trigger

## Description
* As a nurse, I want to trigger comprehensive form review validation by voice command or button click, so that I can verify all fields are correct before submission.

## Acceptance Criteria
1. **Given** all mandatory fields are filled, **When** I say "Please review the form" or click Review button, **Then** the system triggers comprehensive validation across all layers (schema, data type, regex, required fields).

2. **Given** validation executes, **When** the system processes all fields, **Then** the system identifies and categorizes all errors, warnings, and mismatched values by section (Demographics & Admission, Clinical Intake).

3. **Given** validation finds errors, **When** results are ready, **Then** the system verbally summarizes errors via TTS: "I found 2 errors. In Demographics section, Date of Birth format is invalid. In Clinical Intake section, HBsAb value is missing. Would you like to edit these fields?".

4. **Given** validation passes, **When** no errors are found, **Then** the system announces "All fields are valid. Do you want to submit?" via TTS and enables Continue button.

5. **Given** Review button exists, **When** form is populated, **Then** the system displays Review button in enabled state by default and Continue button in disabled state until validation passes.

## Edge Cases
* What happens when validation is triggered with incomplete mandatory fields? (Display error count; announce "3 required fields are incomplete. Please complete all required fields before review.")
* How does the system handle validation timeout? (Set 5-second timeout for validation; display error "Validation timeout. Please try again." if exceeded)
