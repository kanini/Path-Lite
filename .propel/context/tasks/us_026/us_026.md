# User Story - US_026

## Story ID
* ID: US_026

## Story Title
* No-Response Handling

## Description
* As a nurse, I want the system to handle no-response scenarios by repeating questions and auto-closing after 3 failed attempts, so that the session doesn't hang indefinitely when I'm interrupted.

## Acceptance Criteria
1. **Given** AI asks a question, **When** no response is detected for 5 seconds, **Then** the system repeats the question via TTS and reactivates STT listening.

2. **Given** no response is detected again, **When** the second 5-second timeout occurs, **Then** the system repeats the question a second time with additional prompt "Please respond or say 'skip' to continue.".

3. **Given** no response occurs a third time, **When** the third 5-second timeout occurs, **Then** the system announces "No response detected. Saving your progress and closing the session." via TTS.

4. **Given** 3 failed response attempts occur, **When** the session auto-closes, **Then** the system saves partial form data to MMKV with last field index, displays visual notification "Session closed due to inactivity", and returns to Patient Dashboard.

5. **Given** session auto-closes, **When** I return to the treatment page, **Then** the system detects incomplete session and offers resume option "You have an incomplete session. Would you like to resume?".

## Edge Cases
* What happens when background noise triggers false detection? (Implement voice activity detection threshold; only reset timer for actual speech, not ambient noise)
* How does the system handle no-response during mandatory field? (Save partial data; mark field as incomplete; resume from same mandatory field on next session)
