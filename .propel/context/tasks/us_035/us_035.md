# User Story - US_035

## Story ID
* ID: US_035

## Story Title
* Final Submission Workflow

## Description
* As a nurse, I want to submit the completed form with confirmation feedback, so that I know my patient documentation has been saved successfully.

## Acceptance Criteria
1. **Given** I confirm submission, **When** I say "Report looks fine, submit" or tap Submit button, **Then** the system detects final submission intent using LLM classification and initiates submission process.

2. **Given** submission starts, **When** the process begins, **Then** the system displays branded submission progress overlay "Submitting this treatment - Please wait" with percentage progress bar incrementing to 100%.

3. **Given** form data is ready, **When** submission executes, **Then** the system saves mock form data to local MMKV storage (no database) with timestamp, userId, and submission status.

4. **Given** submission completes, **When** data is saved, **Then** the system generates submission confirmation, announces "Form submitted successfully. Patient record has been saved." via TTS, and displays success message.

5. **Given** submission is confirmed, **When** success message displays, **Then** the system redirects to Patient Dashboard after 2 seconds and updates patient status to "Submitted" in the patient list.

## Edge Cases
* What happens when MMKV save fails during submission? (Retry save 3 times; display error "Submission failed. Please try again."; preserve form data for retry)
* How does the system handle submission during network interruption? (Allow local save to MMKV; display success message; sync will occur when connectivity restored in future phases)
