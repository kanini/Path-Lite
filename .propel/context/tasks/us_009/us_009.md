# User Story - US_009

## Story ID
* ID: US_009

## Story Title
* Mock Data Export Utility

## Description
* As a developer, I want a mock data export utility that exports patient records and treatment sessions to JSON format, so that I can validate future EHR integration data structure and facilitate testing.

## Acceptance Criteria
1. **Given** I need to export patient data, **When** I trigger the export function, **Then** the system exports all Patient records from MMKV to a JSON file with proper structure and encryption removed.

2. **Given** I need to export treatment sessions, **When** I trigger the session export, **Then** the system exports all TreatmentSession records with associated patient data in a nested JSON structure.

3. **Given** I need to export audit logs, **When** I trigger the audit export, **Then** the system exports all AuditLog records from the separate MMKV instance to a JSON file with timestamp-based filename.

4. **Given** exported data needs validation, **When** I review the exported JSON, **Then** the system includes schema version, export timestamp, record count, and validates JSON structure before export.

5. **Given** export needs to be accessible, **When** I complete the export, **Then** the system saves the JSON file to device documents directory and provides shareable file path for external transfer.

## Edge Cases
* What happens when MMKV storage is empty during export? (Generate empty JSON array with metadata; display warning "No data available to export")
* How does the system handle large export files exceeding device storage? (Implement size check before export; provide chunked export option for large datasets)
