# User Story - US_031

## Story ID
* ID: US_031

## Story Title
* Field Re-validation After Correction

## Description
* As a developer, I want automatic field re-validation after correction attempts, so that corrected values are verified before accepting them and returning to review.

## Acceptance Criteria
1. **Given** corrected input is received, **When** the nurse provides a new value, **Then** the system immediately validates the corrected value against all validation layers (schema, data type, regex, business rules).

2. **Given** validation passes, **When** the corrected value is valid, **Then** the system updates the field with the new value, announces success via TTS ("Date of Birth updated. Moving to next error."), and navigates to the next erroneous field.

3. **Given** validation fails again, **When** the corrected value is still invalid, **Then** the system announces the specific error via TTS, re-asks the question, and allows up to 3 correction attempts.

4. **Given** 3 correction attempts fail, **When** all retries are exhausted, **Then** the system offers manual entry option: "Would you like to enter this manually?" and provides text input field.

5. **Given** all corrections are complete, **When** the last erroneous field is fixed, **Then** the system automatically returns to Review screen, triggers comprehensive validation again, and announces results.

## Edge Cases
* What happens when correction introduces new validation error? (Announce new error; explain what's wrong; allow correction retry)
* How does the system handle manual entry after voice correction fails? (Switch to text input mode; apply same validation rules; accept manual input if valid)
