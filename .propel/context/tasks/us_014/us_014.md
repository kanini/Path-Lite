# User Story - US_014

## Story ID
* ID: US_014

## Story Title
* Patient Dashboard Implementation

## Description
* As a nurse, I want a Patient Dashboard displaying all mock patients for my selected hospital with Active/Completed status filtering, so that I can view existing patients and initiate new patient entries.

## Acceptance Criteria
1. **Given** I select a hospital, **When** the Patient Dashboard loads, **Then** the system displays two tabs "My Patients" and "All Patients" with patient list showing name, MRN, status badge (Not Started, Tx In Progress, Received, Submitted, Submitted (Amended)), and last updated timestamp.

2. **Given** I view the patient list, **When** I tap on a patient entry, **Then** the system navigates to the treatment page with patient data prefilled from hardcoded mock data.

3. **Given** I need to add a new patient, **When** I tap the "Add New" button, **Then** the system displays treatment type selection dropdown with "Hemodialysis" option and opens Patient Recall Search popup after selection.

4. **Given** I filter patients, **When** I switch between "My Patients" and "All Patients" tabs, **Then** the system filters the patient list to show only patients assigned to me or all patients for the hospital respectively.

5. **Given** I need to refresh data, **When** I pull down on the patient list, **Then** the system reloads patient data from MMKV storage and updates the list with latest status information.

## Edge Cases
* What happens when no patients exist for the hospital? (Display empty state "No patients found. Tap 'Add New' to create a patient."; show Add New button prominently)
* How does the system handle patient status badge colors? (Use consistent color coding: Not Started (gray), Tx In Progress (blue), Received (orange), Submitted (green), Submitted (Amended) (purple))
