# User Story - US_017

## Story ID
* ID: US_017

## Story Title
* Text-to-Speech Integration

## Description
* As a nurse, I want the system to ask form questions using Text-to-Speech, so that I can hear questions without reading the screen and maintain focus on patient care.

## Acceptance Criteria
1. **Given** I need TTS capability, **When** I integrate expo-speech library, **Then** the system has version 11.7+ installed with platform-native synthesis for iOS and Android.

2. **Given** the form opens, **When** TTS activates automatically, **Then** the system speaks the first question "Please provide the Medical Record Number" with clear audio quality and natural voice.

3. **Given** TTS is speaking, **When** I need to configure voice settings, **Then** the system provides options for speech rate (0.5x to 2.0x) and voice selection (male/female) with default settings optimized for clarity.

4. **Given** a question needs to be asked, **When** TTS synthesizes speech, **Then** the system completes synthesis in under 500ms and plays audio through device speakers with adequate volume.

5. **Given** TTS completes a question, **When** speech finishes, **Then** the system automatically activates STT listening mode and displays listening indicator.

## Edge Cases
* What happens when device volume is muted? (Detect muted state; display visual alert "Please unmute device to hear questions" with volume icon)
* How does the system handle TTS interruption by incoming calls? (Pause TTS on interruption; resume from last question when call ends)
