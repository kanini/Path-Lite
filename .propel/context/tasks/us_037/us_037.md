# User Story - US_037

## Story ID
* ID: US_037

## Story Title
* Question Configuration File

## Description
* As a developer, I want a JSON question configuration file with TTS question templates for all 15 form fields, so that the AI can ask properly formatted questions during voice interaction.

## Acceptance Criteria
1. **Given** I need question templates, **When** I create the configuration file, **Then** the system has a JSON file stored locally with TTS question templates for all 15 fields (e.g., "Please provide the Medical Record Number", "What is the patient's first name?").

2. **Given** questions need context, **When** I define each template, **Then** the system includes field name, question text, field type, and optional clarification text for each question.

3. **Given** enum fields need special handling, **When** I configure enum questions, **Then** the system includes available options in the question (e.g., "What is the patient's gender? Male or Female?").

4. **Given** questions need ordering, **When** I structure the file, **Then** the system maps questions to field ordering (1-15) to ensure sequential conversation flow matches schema definition.

5. **Given** questions need variations, **When** I provide alternatives, **Then** the system includes rephrased versions for clarification requests (e.g., original: "Please provide the Medical Record Number", rephrased: "What is the patient's MRN or Medical Record Number?").

## Edge Cases
* What happens when question template is missing for a field? (Use default template "Please provide [field name]"; log warning for missing template)
* How does the system handle localization in future? (Design question file to support multiple languages; use language code as key)
