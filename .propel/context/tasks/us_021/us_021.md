# User Story - US_021

## Story ID
* ID: US_021

## Story Title
* Prompt Engineering for Medical Terms

## Description
* As a developer, I want prompt engineering with few-shot examples for medical terminology extraction, so that the LLM accurately identifies and extracts domain-specific healthcare terms.

## Acceptance Criteria
1. **Given** I need medical term extraction, **When** I design the prompt, **Then** the system includes few-shot examples for common medical terms: HBsAg (Positive/Negative/Unknown), Treatment Location (OR/Bedside/ICU-CCU/ER/Multi-Tx Room), Gender (Male/Female).

2. **Given** I need context-aware extraction, **When** I structure the prompt, **Then** the system includes field context (e.g., "Extract the patient's gender from the following response:") and expected format guidance.

3. **Given** I need consistent outputs, **When** I test the prompt, **Then** the system produces consistent extractions across similar inputs with >95% accuracy for medical terminology.

4. **Given** I need to handle variations, **When** I include examples, **Then** the system handles variations like "Hep B positive" → "Positive", "male patient" → "Male", "operating room" → "OR".

5. **Given** I need validation, **When** I validate LLM outputs, **Then** the system checks extracted values against form schema enums and rejects invalid values with re-prompt request.

## Edge Cases
* What happens when LLM hallucinates medical terms? (Validate against schema enums; reject hallucinated values; re-ask question with clarification)
* How does the system handle abbreviations? (Include abbreviation mappings in prompt; expand "Hep B" to "HBsAg", "OR" to "Operating Room")
