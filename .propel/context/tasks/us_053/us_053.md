# User Story - US_053

## Story ID
* ID: US_053

## Story Title
* LLM Failure Fallback

## Description
* As a nurse, I want the system to fallback to manual entry mode when LLM features fail, so that I can continue data entry even when AI is unavailable.

## Acceptance Criteria
1. **Given** LLM call fails, **When** Azure OpenAI is unavailable or returns error, **Then** the system detects the failure, logs error details, and displays message "AI features temporarily unavailable. Switching to manual entry."

2. **Given** manual entry mode is activated, **When** the fallback occurs, **Then** the system disables voice input, displays text input fields for all form fields, and maintains same validation rules.

3. **Given** I enter data manually, **When** I type in fields, **Then** the system applies client-side validation, provides inline error messages, and saves data to MMKV without requiring LLM processing.

4. **Given** LLM service is restored, **When** connectivity returns, **Then** the system displays notification "AI features restored. Would you like to switch back to voice mode?" with option to continue manual or resume voice.

5. **Given** session state is preserved, **When** fallback occurs mid-session, **Then** the system preserves all previously entered field values, maintains session continuity, and allows seamless transition between voice and manual modes.

## Edge Cases
* What happens when LLM fails for specific fields only? (Allow mixed mode: manual entry for failed fields, voice for successful fields; track mode per field)
* How does the system handle repeated LLM failures? (After 3 consecutive failures, suggest manual mode for remainder of session; log pattern for investigation)
