# User Story - US_038

## Story ID
* ID: US_038

## Story Title
* Responsive Layout Implementation

## Description
* As a nurse, I want the application to adapt to different tablet sizes (7-12.9 inch) in both portrait and landscape orientations, so that I can use the app comfortably on any device.

## Acceptance Criteria
1. **Given** I use a 7-inch tablet, **When** I view the app in portrait mode (600dp width), **Then** the system renders the layout correctly with appropriately scaled touch targets (≥44pt), readable typography (≥14pt body text), and no horizontal scroll.

2. **Given** I use a 12.9-inch tablet, **When** I view the app in landscape mode (1366dp width), **Then** the system uses split-pane layout with left navigation panel and main content panel, optimizing use of expanded screen real estate.

3. **Given** I rotate the device, **When** I switch between portrait and landscape orientations, **Then** the system reflows the layout within 300ms, preserves all entered field values, and maintains scroll position.

4. **Given** I view forms on large tablets, **When** the screen is in landscape mode, **Then** the system displays patient summary and navigation in left panel (30% width) and form content in right panel (70% width).

5. **Given** I view forms on small tablets, **When** the screen is in portrait mode, **Then** the system uses single-column layout with collapsible navigation drawer and full-width form content.

## Edge Cases
* What happens when device is in split-screen mode? (Detect reduced viewport; adapt to smallest supported size; display warning if below 7-inch minimum)
* How does the system handle orientation lock? (Respect device orientation lock settings; maintain current orientation without forcing rotation)
