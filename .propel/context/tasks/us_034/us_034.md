# User Story - US_034

## Story ID
* ID: US_034

## Story Title
* Report Generation

## Description
* As a nurse, I want a patient report generated from submitted form data, so that I can review the complete documentation before final submission.

## Acceptance Criteria
1. **Given** validation passes, **When** I tap Continue button, **Then** the system asks "Do you want to continue?" via TTS and displays View Report and Submit buttons.

2. **Given** I tap View Report, **When** the button is clicked, **Then** the system generates a formatted report from mock form data with patient demographics, admission details, and clinical intake information organized in sections.

3. **Given** report is generated, **When** the report displays, **Then** the system shows all field labels and values in a structured document format with clear section headers and proper formatting.

4. **Given** I review the report, **When** I identify an error, **Then** the system provides "Go back" or "Edit form" commands that return to form editing mode and allow corrections.

5. **Given** report looks correct, **When** I'm satisfied with the data, **Then** the system waits for user instruction and detects final submission intent when I say "Report looks fine, submit".

## Edge Cases
* What happens when report generation fails? (Display error "Unable to generate report. Please try again."; provide retry button)
* How does the system handle missing optional fields in report? (Display "Not provided" for empty optional fields; maintain report structure)
