# Build Agent Orchestrator

**Target Persona**: Software Developers, Full Stack Engineers

Orchestrates the implementation phase: development task planning, task implementation with analysis and reviews. For user stories, also includes unit test planning and implementation. Consumes user stories `/backlog-agent` or direct input.

**Prerequisite**: For user stories, run `/backlog-agent` first to generate user stories. 

**Handoff**: After successful test suite run and code review completion, address any findings before PR/deployment.

## Input Parameter: $ARGUMENTS (Mandatory)
**Accepts:** User Story ID | User Story file path
**Supported Patterns:**
- User Story: `us_XXX` | `US-XXX` | `.propel/context/tasks/us_XXX/us_XXX.md`

### Input Processing Instructions
**CRITICAL**: Before proceeding with orchestration, you MUST determine input type and process accordingly:

#### Input Type Detection
1. **User Story Detection**: Check if `$ARGUMENTS` matches:
   - Pattern: `us_\d{3}` or `US[-_]?\d{3,4}`
   - File path containing `us_XXX`
   - Text starting with "As a..." (user story format)

#### AI Task Detection (After Input Validation)

Read user story file
Grep("AIR-\d{3}", user_story_file) → air_requirements
IF air_requirements.count > 0:
  Set AI_IMPACT = true
  Print "AI requirements detected (AIR-XXX). AI-specific validation will be applied."
```

**AI Impact Implications:**
- Plan Development Tasks will generate AI layer tasks (prompt setup, RAG pipeline, guardrails)
- Implementation will load AI context (prompt templates, guardrails config)
- Analysis will include AIR-XXX requirement validation
- Code Review will include AI security checklist

#### Input Validation
- If User Story ID: Verify story exists in `.propel/context/tasks/us_XXX/us_XXX.md`
- If file path: Verify file exists and is readable
- If text input: Parse and extract relevant identifiers

## Workflow Steps

### Feature Path
1. **Plan Development Tasks** - Generate implementation tasks from user story
   - For each task:
     - 1.1. **Implement Task** - Execute the development task
     - 1.2. **Analyze Implementation** - Review implementation against requirements
     - 1.3. **Analyze UX** (CONDITIONAL) - Evaluate UI/UX if task has UI impact
     - 1.4. **Review Code** - Comprehensive code review
2. **Plan Unit Tests** - Generate unit test plan for the user story
3. **Implement Unit Tests** - Implement and Execute the unit tests as per plan

## Output
- Artifacts generated by each sub-workflow:
  - Step 1 : `.propel/context/tasks/us_XXX/task_*.md`
  - Step 1.2: `.propel/context/tasks/us_XXX/reviews/task-review-*.md`
  - Step 1.3 (if UI): `.propel/context/tasks/us_XXX/reviews/ui-review-*.md`
  - Step 1.4: `.propel/code-reviews/review_*.md`
  - Step 2 : `.propel/context/tasks/us_XXX/unittest/test_plan_*.md`
  - Step 3.2 : `.propel/context/tasks/us_XXX/reviews/task-review-*.md` (for test implementation)
- Print progress after each step (e.g., "Step 1/3 completed", "Task 1.1/1.5 completed")
- Print key findings summary for each completed step
- Print final recommendation to review PR and merge

## Orchestration Instructions

When invoked, execute the following workflow based on detected input type:

---

### Step 1: Plan Development Tasks
- **Invoke**: `.windsurf/workflows/plan-development-tasks.md` with `$ARGUMENTS`
- **Input**: User story file or ID
- **Wait**: Allow workflow to complete fully
- **Capture**: List of generated task files for iteration
- **Summarize**: Present key findings including:
  - Number of tasks generated (task_XXX)
  - Technology layers covered (FE/BE/DB/Infra)
  - Estimated total effort
  - Tasks with UI impact (for conditional UX analysis)
  - Acceptance criteria coverage
- **Progress**: Print "Step 1/3 started: Plan Development Tasks"
- **Next**: Proceed to Task Implementation Loop

---

### Task Implementation Loop (For Each Generated Task)

For each task file generated in Step 1, execute the following sub-steps sequentially:

#### Step 1.1: Implement Task
- **Invoke**: `.windsurf/workflows/implement-tasks.md` with current task file path
- **Input**: Task file from `.propel/context/tasks/us_XXX/task_*.md`
- **Wait**: Allow workflow to complete fully
- **Summarize**: Present key findings including:
  - Files created/modified
  - Requirements implemented
  - Build and test status
  - Todo items completed
- **Progress**: Print "Task [N]/[Total] - Step 1.1 completed: Implement Task"

#### Step 1.2: Analyze Implementation
- **Invoke**: `.windsurf/workflows/analyze-implementation.md` with current task file path
- **Input**: Same task file used in Step 1.1
- **Wait**: Allow workflow to complete fully
- **Purpose**: Verifies that the **TODOs defined in the task file** were implemented correctly
  - Each task file contains specific TODOs derived from the parent (user story AC)
  - This step checks task-level completion, NOT user story AC directly
- **Summarize**: Present key findings including:
  - TODO completion status (completed vs pending)
  - Requirements fulfillment percentage (task-level)
  - Gap analysis (missing implementations)
  - Code quality assessment
  - Recommended actions for any gaps
- **Progress**: Print "Task [N]/[Total] - Step 1.2 completed: Analyze Implementation"

#### Step 1.3: Analyze UX (Feature Path Only - CONDITIONAL)
- **UI Impact Gate**: Check task file's "Design References" table for `UI Impact` value:
  - If `UI Impact = Yes`: Proceed with UX analysis
  - If `UI Impact = No` or table missing: Skip UX analysis
- **IF UI impact detected**:
  - **Invoke**: `.windsurf/workflows/analyze-ux.md` with parameters:
    - `task_file`: Current task file path (e.g., `.propel/context/tasks/us_XXX/task_001_*.md`)
      - Workflow will automatically extract: Figma URL, Hi-Fi Wireframe path, Screen ID, UXR Requirements
    - `server_url`: Development server URL (default: http://localhost:3000)
  - **Wait**: Allow workflow to complete fully
  - **Summarize**: Present key findings including:
    - Visual consistency score (design token deviations)
    - Hi-Fi wireframe comparison results (pixel deviations documented)
    - State coverage score ([N]/5 states implemented)
    - Accessibility compliance (WCAG 2.1 AA)
    - Responsive design validation (desktop/tablet/mobile)
    - Interaction quality assessment (hover/focus/active states)
    - UXR requirements validation checklist
  - **Progress**: Print "Task [N]/[Total] - Step 1.3 completed: Analyze UX"
- **IF NO UI impact detected**:
  - Print "No UI impact detected in task (UI Impact = No). Skipping UX analysis."
  - **Progress**: Print "Task [N]/[Total] - Step 1.3 skipped: No UI impact"

#### Step 1.4: Review Code
- **Invoke**: `.windsurf/workflows/review-code.md`
- **Input**: Reviews local changes (files modified during implementation)
- **Wait**: Allow workflow to complete fully
- **Summarize**: Present key findings including:
  - Files reviewed count
  - Findings by severity (Critical/High/Medium/Low)
  - Security coverage (OWASP checks)
  - Architecture assessment
  - Recommendation (Approve/Approve with Changes/Reject)
- **Progress**:
  - "Task [N]/[Total] - Step 1.4 completed: Review Code"
- **Gate**: If Critical findings exist, pause and ask user how to proceed

#### Task Loop Completion
- After completing all sub-steps for current task:
  - Print "Task [N]/[Total] completed successfully"
  - Proceed to next task in the list
- After all tasks complete:
  - Print "All implementation tasks completed"

---

### Step 2: Plan Unit Tests 
- **Invoke**: `.windsurf/workflows/plan-unit-test.md` with original `$ARGUMENTS` (User Story ID/path)
- **Input**: User story file from `.propel/context/tasks/us_XXX/us_XXX.md`
- **Wait**: Allow workflow to complete fully
- **Capture**: List of generated test plan files for Step 3
- **Summarize**: Present key findings including:
  - Number of test plans generated by layer (FE/BE/DB)
  - Total test cases designed
  - Acceptance criteria coverage
  - Mock requirements identified
  - Expected code coverage percentage
- **Progress**: Print "Step 2/3 completed: Plan Unit Tests"

---

### Step 3: Implement Unit Tests
- **For each test plan file** generated in Step 2:

  #### Step 3.1: Implement Test Plan
  - **Invoke**: `.windsurf/workflows/implement-tasks.md` with test plan file path
  - **Input**: Test plan file from `.propel/context/tasks/us_XXX/unittest/test_plan_*.md`
  - **Wait**: Allow workflow to complete fully
  - **Summarize**: Present key findings including:
    - Test files created
    - Test cases implemented
    - Coverage achieved
    - All tests passing status
  - **Progress**: Print "Step 3/3 - Test Plan [N]/[Total] - Step 3.1 completed: Implement Test Plan"

  #### Step 3.2: Analyze Test Implementation
  - **Invoke**: `.windsurf/workflows/analyze-implementation.md` with test plan file path
  - **Input**: Same test plan file used in Step 3.1
  - **Wait**: Allow workflow to complete fully
  - **Summarize**: Present key findings including:
    - Todo completion status (all test plan todos implemented)
    - Test cases implemented vs planned
    - Gaps identified (missing test cases or coverage)
    - Requirements traceability verification
    - Recommended actions for any gaps
  - **Progress**: Print "Step 3/3 - Test Plan [N]/[Total] - Step 3.2 completed: Analyze Test Implementation"

- **After all test plans implemented and analyzed**:
  - Run full test suite to verify coverage
  - Print overall coverage statistics
  - **Progress**: Print "Step 3/3 completed: Implement Unit Tests"

---

### Final Summary
After all steps complete:

- Present overall implementation completion summary
- Report workflow path executed (3-step Feature workflow)
- List all artifacts generated with file paths:
  - Development tasks: `.propel/context/tasks/us_XXX/task_*.md`
  - Implementation reviews: `.propel/context/tasks/us_XXX/reviews/task-review-*.md`
  - UI reviews (if applicable): `.propel/context/tasks/us_XXX/reviews/ui-review-*.md`
  - Code reviews: `.propel/code-reviews/review_*.md`
  - Unit test plans: `.propel/context/tasks/us_XXX/unittest/test_plan_*.md`
  - Unit test files: Created in codebase per test plan
  - Test implementation reviews: `.propel/context/tasks/us_XXX/reviews/task-review-*.md`
- Report total test coverage achieved
- Highlight any [UNCLEAR] items or failed gates requiring attention
- Provide recommendation: "Review code review findings and run full regression tests before deployment."

## Execution Rules

1. **Sequential Execution**: Complete each step and sub-step fully before proceeding to the next
2. **Conditional UX Analysis**: Step 1.3 (Analyze UX) executes only for Feature Path and only if UI impact detected in task
4. **Task Loop**: For each generated task, execute all sub-steps sequentially
6. **State Preservation**: All results are saved to `.propel/context/` directories
7. **Explicit Context**: Each workflow references previous step outputs explicitly
8. **Error Handling**: If a step fails, stop execution and ask user how to proceed
9. **Critical Gate**: If code review finds Critical severity issues, pause for user decision
10. **No Quality Gates at Orchestrator Level**: Quality evaluation is handled by individual sub-workflows
