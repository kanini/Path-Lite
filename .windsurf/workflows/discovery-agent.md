# Discovery Agent Orchestrator

**Target Persona**: Solution Architects, System Architects, Product Architects

Orchestrates technical discovery phase: requirements analysis, architecture design, system modeling, UX specification (if UI impact), test strategy, and project plan. Produces technical specifications and project governance artifacts that enable backlog construction.

**Handoff**: Run `/backlog-agent` to transform specs into epics and user stories.

## Input Parameters

### $ARGUMENTS (Mandatory) - Project Scope
**Accepts:** Task description | Project scope file | Requirements document
**Supported File Types:** .pdf | .txt | .md | .docx

### $AI_TRIAGE (Optional) - AI Suitability Assessment
**Default:** `true`
- `true` - Enable AI suitability triage in Step 1, tag requirements with `[AI-CANDIDATE]`, `[DETERMINISTIC]`, `[HYBRID]`
- `false` - Skip AI triage for pure deterministic projects. All downstream workflows treat project as deterministic.

**Note:** This parameter is passed to Step 1 (create-spec.md). When `false`, design-architecture.md will detect no AI tags and skip AI-related sections automatically.

### Input Processing Instructions
**CRITICAL**: Before proceeding with orchestration, you MUST determine input type and process accordingly:

#### Input Type Detection
1. **File Path Detection**: Check if `$ARGUMENTS` contains a file path (contains file extensions .pdf, .txt, .md, .docx)
2. **Direct Text Detection**: If `$ARGUMENTS` doesn't contain a file extension, treat it as direct task description

#### Input Validation
- If file path: Verify file exists and is readable
- If direct text: Ensure sufficient detail for requirements generation
- If no input provided: Request project scope file or task description

## Workflow Steps
This orchestrator executes technical discovery sequentially:

1. **Discover Requirements** - Create functional requirements and use cases
2. **Design Architecture** - Create NFR, DR, TR requirements
3. **System Modeling** - Create Architectural Diagrams
4. **Create Figma Spec** (CONDITIONAL) - Generate UXR-XXX if UI impact detected
5. **Define Test Strategy** - Create E2E functional test plan
6. **Create Project Plan** - Generate project plan with scope, milestones, cost baseline, risks

## Output
- Artifacts generated by each sub-workflow:
  - Step 1: `.propel/context/docs/spec.md`
  - Step 2: `.propel/context/docs/design.md`
  - Step 3: `.propel/context/docs/models.md`
  - Step 4 (if UI): `.propel/context/docs/figma_spec.md`, `.propel/context/docs/designsystem.md`
  - Step 5: `.propel/context/docs/test_plan_[feature].md`
  - Step 6: `.propel/context/docs/project_plan.md`
- Print progress after each step (e.g., "Step 1/6 completed", "Step 6/6 completed")
- Print key findings summary for each completed step
- Print final recommendation to review outcomes and run `/backlog-agent`

## Orchestration Instructions

When invoked, execute the following workflow sequentially:

### Step 1: Discover Requirements
- **Invoke**: `.windsurf/workflows/create-spec.md` with the user's task description or project scope file (`$ARGUMENTS`)
- **Pass Parameter**: `$AI_TRIAGE` (defaults to `true` if not specified)
- **Wait**: Allow workflow to complete fully
- **Summarize**: Present key findings including:
  - Number of FR-XXX requirements generated
  - Number of UC-XXX use cases documented
  - AI Classification Tags summary (if `$AI_TRIAGE=true`):
    - Count of `[AI-CANDIDATE]` tagged requirements
    - Count of `[DETERMINISTIC]` tagged requirements
    - Count of `[HYBRID]` tagged requirements
  - Number of [UNCLEAR] tagged items (requiring clarification)
  - Key business objectives identified
  - Notable risks or constraints
- **AI Triage Note**: If `$AI_TRIAGE=false`, print "AI triage skipped. Project treated as deterministic."
- **Progress**: Print "Step 1/6 completed: Discover Requirements"

### Step 2: Design Architecture
- **Invoke**: `.windsurf/workflows/design-architecture.md`
- **Input**: References `.propel/context/docs/spec.md` automatically (reads AI tags from Step 1)
- **Wait**: Allow workflow to complete fully
- **Summarize**: Present key findings including:
  - Number of NFR-XXX requirements generated
  - Number of TR-XXX requirements generated
  - Number of DR-XXX requirements generated
  - AI Consideration status:
    - If applicable: Number of AIR-XXX requirements generated, AI Architecture Pattern selected
    - If not applicable: "AI Consideration: Not applicable (deterministic project)"
  - Number of [UNCLEAR] tagged items
  - Technology stack decisions (including AI/ML stack if AIR-XXX present)
  - Key architectural patterns selected
- **Progress**: Print "Step 2/6 completed: Design Architecture"

### Step 3: System Modeling
- **Invoke**: `.windsurf/workflows/design-model.md`
- **Input**: References `.propel/context/docs/spec.md` and `.propel/context/docs/design.md` automatically
- **Wait**: Allow workflow to complete fully
- **Summarize**: Present key findings including:
  - Architectural diagrams generated (Context, Component, Deployment, Data Flow, ERD)
  - Number of sequence diagrams created (one per UC-XXX)
  - AI Architecture Diagrams (if AIR-XXX present in design.md):
    - RAG Pipeline Diagram (if RAG/Hybrid pattern)
    - AI Sequence Diagrams (per AI-enabled UC-XXX)
  - Number of [UNCLEAR] tagged items
  - Key architectural views documented
- **Progress**: Print "Step 3/6 completed: System Modeling"

### Step 4: Create Figma Spec (CONDITIONAL)
- **UI Impact Gate**: Scan `spec.md` for UI-related requirements:
  - FR-XXX containing: "display", "screen", "form", "UI", "interface", "dashboard", "page", "view"
  - UC-XXX with Actor = "User" AND visual interaction steps
  - Technology mentions: React, Angular, Vue, Flutter, iOS, Android, frontend
- **IF UI impact detected**:
  - **Invoke**: `.windsurf/workflows/create-figma-spec.md`
  - **Input**: References `.propel/context/docs/spec.md`
  - **Wait**: Allow workflow to complete fully
  - **Summarize**: Present key findings including:
    - Number of UXR-XXX requirements generated
    - Number of SCR-XXX screens identified
    - Number of FL-XXX flows documented
    - Number of [UNCLEAR] tagged items
    - Design system tokens created
  - **Progress**: Print "Step 4/6 completed: Create Figma Spec"
- **IF NO UI impact detected**:
  - Print "No UI impact detected in spec.md. Skipping Figma specification."
  - **Progress**: Print "Step 4/6 skipped: No UI impact"

### Step 5: Define Test Strategy
- **Invoke**: `.windsurf/workflows/create-test-plan.md`
- **Input**: References `.propel/context/docs/spec.md` and `.propel/context/docs/design.md` automatically
- **Wait**: Allow workflow to complete fully
- **Summarize**: Present key findings including:
  - Number of test cases generated
  - FR coverage percentage
  - UC coverage percentage
  - NFR validation approach
  - E2E journey coverage
  - Number of [UNCLEAR] tagged items
- **Progress**: Print "Step 5/6 completed: Define Test Strategy"

### Step 6: Create Project Plan
- **Invoke**: `.windsurf/workflows/create-project-plan.md`
- **Input**: References `.propel/context/docs/spec.md`, `.propel/context/docs/design.md`, `.propel/context/docs/models.md`, and `.propel/context/docs/figma_spec.md` (if exists) automatically
- **Wait**: Allow workflow to complete fully
- **Summarize**: Present key findings including:
  - Number of milestones generated (MS-XXX)
  - Number of risks identified (RK-XXX)
  - Estimated project duration (weeks)
  - Cost baseline summary
  - Key risk categories covered (Technical, Resource, Schedule, Scope)
- **Progress**: Print "Step 6/6 completed: Create Project Plan"

- List all artifacts generated with file paths
- **AI Status Summary**:
  - If AI-enabled project: List AIR-XXX count, AI Pattern selected, AI/ML stack
  - If deterministic project: "AI Consideration: Not applicable"
- **Project Plan Summary**:
  - Milestone count (MS-XXX)
  - Risk count (RK-XXX)
  - Estimated duration
- **Total [UNCLEAR] Count**: Sum of all [UNCLEAR] tagged items across all steps (requires manual resolution before backlog building)
- Provide recommendation: "Please review the generated artifacts in `.propel/context/docs/` directory"
- Highlight specific [UNCLEAR] items requiring manual clarification
- **Next Step**: Recommend running `/backlog-agent` to build epics, user stories, and sprint plans

## Execution Rules

1. **Sequential Execution**: Complete each step fully before proceeding to the next
2. **Conditional Step**: Step 4 (Create Figma Spec) executes only if UI impact detected in spec.md
3. **Dependency Chain**: Each step depends on all previous steps' outputs
4. **State Preservation**: All results are saved to `.propel/context/docs/`
5. **Explicit Context**: Each workflow references previous step outputs explicitly
6. **Error Handling**: If a step fails, stop execution and ask user how to proceed
7. **No Quality Gates**: Quality evaluation is handled by individual sub-workflows; this orchestrator does not enforce additional quality criteria

## Usage
**Invoke with**: `/discovery-agent [detailed task description]` | `/discovery-agent [*.md|*.docx|*.pdf]`

**Optional Parameter**: `$AI_TRIAGE=true|false` (default: `true`)

**Examples**:
- `/discovery-agent Project_Scope.md` (AI triage enabled by default)
- `/discovery-agent Project_Scope.md $AI_TRIAGE=false` (Skip AI triage for pure deterministic project)
- `/discovery-agent Build a task management application with user authentication, project boards, and team collaboration features`
- `/discovery-agent Build a customer support chatbot that answers questions from knowledge base` (AI triage will detect AI-fit)
- `/discovery-agent requirements.pdf`

**Next Step After Completion**: Run `/backlog-agent` to generate epics, wireframes (if figma_spec.md exists), and user stories.
