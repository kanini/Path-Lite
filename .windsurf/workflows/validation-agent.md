# Validation Agent Orchestrator

**Target Persona**: QA Engineers, Test Automation Engineers, Release Managers

Orchestrates the test automation pipeline: converts test plans into automation specifications, generates Playwright TypeScript scripts, executes tests, and produces validation reports. Handles both feature-level and E2E test automation.

**Prerequisite**: Run `/discovery-agent` to generate `test_plan_[feature].md`, OR have `spec.md` available for direct automation test generation.

**Handoff**: After successful test execution, proceed to `/devops-agent` for deployment.

## Input Parameters

### $ARGUMENTS (Optional)
**Accepts:** Test plan path | spec.md path | Empty (auto-detect)
**Default:** Auto-discovers `.propel/context/docs/test_plan_*.md` or `.propel/context/docs/spec.md`

### --type (Optional)
**Accepts:** `feature` | `e2e` | `both`
**Default:** `both`

| Value | Behavior |
|-------|----------|
| `feature` | Generate and execute feature-level tests only (HP, EC, ER) |
| `e2e` | Generate and execute E2E journey tests only |
| `both` | Generate and execute all test types (default) |

### --execute (Optional)
**Accepts:** `true` | `false`
**Default:** `false`

| Value | Behavior |
|-------|----------|
| `true` | Generate scripts AND execute tests |
| `false` | Generate scripts only, skip execution (default) |

### --scope (Optional)
**Accepts:** `full` | `smoke` | `regression`
**Default:** `full`

| Value | Behavior |
|-------|----------|
| `full` | Execute all generated tests |
| `smoke` | Execute P0 critical path tests only |
| `regression` | Execute stable, baselined tests |

### Input Processing Instructions
**CRITICAL**: Before proceeding with orchestration, you MUST determine input type and process accordingly:

#### Input Type Detection
1. **Test Plan Path**: Check if `$ARGUMENTS` contains `test_plan_*.md`
2. **Spec Path**: Check if `$ARGUMENTS` contains `spec.md`
3. **Auto-Detect**: If empty, scan for test_plan files first, then spec.md

#### Input Validation
- If test plan path: Verify file exists in `.propel/context/docs/`
- If spec.md path: Verify file exists
- If auto-detect: Verify at least one source file found
- If no input found: Request user to run `/discovery-agent` first or provide spec.md path

## Workflow Steps

This orchestrator executes test automation sequentially:

1. **Create Automation Tests** - Convert requirements/test plans to automation specs
2. **Generate Playwright Scripts** - Generate TypeScript test files from specs
3. **Execute Tests** (CONDITIONAL) - Run Playwright test suite if `--execute=true`
4. **Generate Report** - Produce validation summary with coverage and results

## Output

- Artifacts generated by each sub-workflow:
  - Step 1: `.propel/context/test/tw_<feature>.md`, `.propel/context/test/e2e_<journey>.md`
  - Step 2: `test-automation/tests/*.spec.ts`, `test-automation/e2e/*.spec.ts`, `test-automation/pages/*.page.ts`
  - Step 3 (if execute): `test-automation/reports/` (HTML report, JUnit XML)
  - Step 4: Console summary with coverage metrics
- Print progress after each step (e.g., "Step 1/4 completed", "Step 4/4 completed")
- Print key findings summary for each completed step
- Print final recommendation based on test results

## Orchestration Instructions

When invoked, execute the following workflow sequentially:

### Step 1: Create Automation Tests
- **Invoke**: `.windsurf/workflows/create-automation-test.md` with source file path
- **Pass Parameter**: `--type` from orchestrator input (default: `both`)
- **Input**:
  - If test_plan provided: Reference for test case structure
  - If spec.md: Direct requirement analysis
- **Wait**: Allow workflow to complete fully
- **Capture**: List of generated test workflow files (tw_*.md, e2e_*.md)
- **Summarize**: Present key findings including:
  - Number of feature test workflows generated (tw_*.md)
  - Number of E2E journey workflows generated (e2e_*.md)
  - Test case count by type (HP, EC, ER)
  - UC coverage percentage
  - Selector strategy summary
- **Progress**: Print "Step 1/4 completed: Create Automation Tests"

---

### Step 2: Generate Playwright Scripts
- **Invoke**: `.windsurf/workflows/generate-playwright-scripts.md`
- **Pass Parameter**: `--type` from orchestrator input
- **Input**: Auto-discovers generated files from Step 1
- **Wait**: Allow workflow to complete fully
- **Capture**: List of generated test files and page objects
- **Summarize**: Present key findings including:
  - Number of spec files generated (*.spec.ts)
  - Number of page objects generated (*.page.ts)
  - TypeScript compilation status
  - Test isolation validation
  - Locator quality assessment (role-based vs CSS)
- **Progress**: Print "Step 2/4 completed: Generate Playwright Scripts"
- **Gate**: If TypeScript compilation fails, pause and ask user how to proceed

---

### Step 3: Execute Tests (CONDITIONAL)
- **Execute Gate**: Check `--execute` parameter:
  - If `--execute=true`: Proceed with test execution
  - If `--execute=false` (default): Skip to Step 4
- **IF execution enabled**:
  - **Scope Filtering**: Apply `--scope` parameter
    - `full`: Run all tests
    - `smoke`: Run with `--grep @smoke` or P0 tagged tests
    - `regression`: Run with `--grep @regression` tag
  - **Execute Command**:
    ```bash
    npx playwright test --reporter=html,junit
    ```
  - **Wait**: Allow test execution to complete
  - **Capture**: Test results, coverage data, failure details
  - **Summarize**: Present key findings including:
    - Total tests executed
    - Pass/Fail/Skip counts
    - Failure details with stack traces
    - Execution duration
    - Screenshots/traces for failures
  - **Progress**: Print "Step 3/4 completed: Execute Tests"
  - **Gate**: If critical failures (>20% fail rate), pause and ask user how to proceed
- **IF execution disabled** (default):
  - Print "Test execution skipped (--execute=false, default). Scripts generated but not executed."
  - Print "To execute tests, re-run with: /validation-agent --execute=true"
  - **Progress**: Print "Step 3/4 skipped: Execution disabled (default)"

---

### Step 4: Generate Report
- **Compile Validation Summary**:
  - Aggregate results from all previous steps
  - Calculate overall coverage metrics
  - Identify gaps and recommendations
- **Report Contents**:
  - Test Generation Summary:
    - Automation specs created (tw_*.md, e2e_*.md)
    - Playwright scripts generated (*.spec.ts)
    - Page objects created (*.page.ts)
  - Execution Summary (if executed):
    - Total: [N] tests | Passed: [N] | Failed: [N] | Skipped: [N]
    - Pass Rate: [X]%
    - Duration: [X]s
  - Coverage Analysis:
    - UC Coverage: [X]% of Use Cases have automation
    - FR Coverage: [X]% of Functional Requirements tested
    - E2E Journey Coverage: [X]% of identified journeys automated
  - Quality Assessment:
    - Locator Quality: [X]% role-based selectors
    - Test Isolation: PASS/FAIL
    - POM Compliance: PASS/FAIL
- **Verdict**:
  - **PASS**: All tests passed, coverage ≥80%
  - **CONDITIONAL PASS**: Minor failures, coverage ≥70%
  - **FAIL**: Critical failures or coverage <70%
- **Progress**: Print "Step 4/4 completed: Generate Report"

---

### Final Summary
After all steps complete:
- Present overall validation completion summary
- List all artifacts generated with file paths:
  - Automation specs: `.propel/context/test/tw_*.md`, `.propel/context/test/e2e_*.md`
  - Test scripts: `test-automation/tests/*.spec.ts`, `test-automation/e2e/*.spec.ts`
  - Page objects: `test-automation/pages/*.page.ts`
  - Test data: `test-automation/data/*.json`
  - Reports: `test-automation/reports/` (if executed)
- Report overall verdict (PASS/CONDITIONAL PASS/FAIL)
- Highlight any failures requiring attention
- **Next Step Recommendation**:
  - If PASS: "Validation complete. Proceed to `/pull-request` or `/devops-agent`"
  - If CONDITIONAL PASS: "Review failures before proceeding. Consider fixing or documenting known issues."
  - If FAIL: "Critical failures detected. Address test failures before proceeding."

## Execution Rules

1. **Sequential Execution**: Complete each step fully before proceeding to the next
2. **Conditional Execution**: Step 3 (Execute Tests) runs only if `--execute=true` (default is `false`)
3. **Dependency Chain**: Each step depends on previous step outputs
4. **State Preservation**: All results are saved to `.propel/context/test/` and `test-automation/`
5. **Explicit Context**: Each workflow references previous step outputs explicitly
6. **Error Handling**: If a step fails, stop execution and ask user how to proceed
7. **Compilation Gate**: If TypeScript fails to compile in Step 2, halt and report
8. **Failure Gate**: If test failure rate >20% in Step 3, pause for user decision
9. **No Quality Gates at Orchestrator Level**: Quality evaluation is handled by individual sub-workflows

## Usage
**Invoke with**: `/validation-agent` | `/validation-agent [test_plan_path]` | `/validation-agent [spec.md path]`

**Optional Parameters**: `--type`, `--execute`, `--scope`

**Examples**:
- `/validation-agent` (auto-detect test plan or spec.md, generate scripts only)
- `/validation-agent --execute=true` (generate scripts AND execute tests)
- `/validation-agent --type feature` (feature tests only)
- `/validation-agent --type e2e --execute=true --scope smoke` (E2E smoke tests with execution)
- `/validation-agent .propel/context/docs/test_plan_login.md` (specific test plan)
- `/validation-agent .propel/context/docs/spec.md --type both` (from spec directly)

**Next Step After Completion**: Based on verdict, proceed to `/pull-request` or `/devops-agent`.
